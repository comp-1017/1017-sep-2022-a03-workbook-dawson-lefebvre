Create navigational links within a website

There are several techniques to navigate within a website. The semantic tag for navigation is the <nav> tag. You can navigate through your website by using realitive addressing. For our examples, we will use the unordered list tag (<ul>).

  • Page to page within the same folder
  • Page to page between folder
  • Staying on the current page
  • Going to a specific place on the current page (On page navigation)

Navigational menus are placed in your <header> semantic area. They may also be placed in your <footer> area as a sitemap. The nav tag, like other semantic tags, should have an <hx> tag. HOWEVER, this header is optional. If you omit the header tag you will receive a document outliner error: Untitled Section. Untill you are shown how to avoid this error message, it will be acceptable on document outline

CSS Coding Styles Summary

Browser Defaults (no CSS) You have no control over the styling
Inline Coding This is for a single specific element (tag) instance. Your CSS will be specified in a parameter within the element. This CSS affects only the element tag that contains the specification. <pstyle = "rule; [rule; ...]">
CONTENT
</p>		 style="color:red; width:50px;"
Embedded The specification of css as embedded happens in your <head> area. This CSS affects all similar elements on the webpage where the CSS is coded. <style>
selector{rule; [rule; ...]}
[selector{rule; [rule; ...]}]
</style>		 <style>p{color:red; width:50px;} h2{color:red; width:50px;}</style>
Linked (Stylesheet) Preferred method of doing CSS. You place your CSS in a seperate physical file. This file can then be used on ANY webpage in your website. The link to this file will be done in your <head> area. The CSS in this file is similar to embedded, EXCEPT there is no style tag. selector{rule; [rule; ...]}
[selector{rule; [rule; ...]}]		 p{color:red; width:50px;} h2{color:red; width:50px;}
